# Приложение для заучивания китайских слов

## Общая философия дизайна

Лучшие приложения для изучения языков создают ощущение спокойной сосредоточенности — никаких лишних элементов, только ты и слово. Для китайского это особенно важно: иероглиф сам по себе визуально богат, и интерфейс должен ему не мешать, а служить фоном.

Стиль — минималистичный, с намёком на восточную эстетику: много белого пространства, тонкие линии, акцентный цвет (глубокий красный #C0392B или чернильный синий #1A237E), шрифт с характером для латиницы и качественный CJK-шрифт для иероглифов (Noto Serif SC или Ma Shan Zheng).

---

## Экраны и логика

**Главный экран** — это дашборд, который встречает тебя каждый день. Сверху — "полоса огня" (streak), снизу — три плитки: _Повторить сегодня_ (N слов по интервальному повторению), _Учить новые_ и _Тренировки_. Никакого перегруза — только то, что нужно сделать сейчас.

**Карточка слова** — сердце приложения. Она должна показывать иероглиф огромно (120–180px), под ним — пиньинь с тонами, выделенными цветом (каждый тон — свой цвет, это классика). При перевороте карточки — русский перевод, пример предложения и кнопка воспроизведения произношения. Анимация переворота — плавная, 3D-вращение.

**Режим тренировки** включает несколько типов заданий, которые чередуются автоматически: выбор перевода из 4 вариантов, написание пиньиня, распознавание тона по звуку, и — для продвинутых — рисование иероглифа пальцем на холсте с проверкой порядка черт.

---

## Ключевые UX-решения

**Интервальное повторение (SRS)** — алгоритм как в Anki: слово появляется чаще, пока не усвоено, потом реже. После ответа пользователь нажимает одну из трёх кнопок: _Не помню_, _Помню_, _Легко_ — и система сама рассчитывает следующий показ.

**Тоны через цвет** — это самая важная педагогическая фишка. Первый тон — синий, второй — зелёный, третий — оранжевый, четвёртый — красный, нейтральный — серый. Везде, где появляется пиньинь, тоны раскрашены. Это работает на уровне визуальной памяти.

**Прогресс через анимацию** — после завершения сессии экран не просто говорит "готово", а показывает маленький праздник: конфетти, анимированный счётчик XP, фраза на китайском с переводом ("很好！— Очень хорошо!").

**Тёмная тема** — обязательна, особенно для вечерних сессий. Фон не чёрный, а тёмно-серый (#1C1C1E), иероглиф белый, акценты — приглушённее.

---

## Структура коллекции

Слова организованы в **тематические наборы** (еда, транспорт, числа, HSK 1–6) и **личные списки**. Можно быстро добавить новое слово прямо из режима изучения. У каждого слова — карточка с этимологией, мнемоникой и примерами из реальных предложений. Мнемоника особенно важна для китайского: "嗎 (ma) — вопросительная частица, похожа на рот (口) с лошадью (馬) — лошадь задаёт вопрос".

---

## Что делает его по-настоящему эффективным

Эффективность достигается не количеством функций, а их правильным сочетанием: SRS-алгоритм + звук произношения + цветовое кодирование тонов + рукописный ввод + короткие сессии (5–10 минут, не больше). Исследования показывают, что частые короткие сессии работают лучше редких длинных — поэтому интерфейс должен подталкивать к этому, а не пытаться удержать пользователя как можно дольше.


# Стек и архитектура приложения

## Frontend

Вот подробное текстовое описание архитектуры. Схема разделена на шесть основных блоков:

### Клиент

Отвечает за пользовательский интерфейс и локальное управление данными.

- **React Native:** Мобильные приложения для iOS и Android.
- **React (PWA):** Веб-версия приложения.
- **Zustand:** Инструмент для управления стейтом (состоянием) приложения.
- **React Query:** Отвечает за кэширование и синхронизацию данных.

_Примечание: Взаимодействие между клиентом и сервером осуществляется по зашифрованному протоколу **HTTPS**._

### Сервер

Центральный узел, обрабатывающий бизнес-логику и запросы.

- **Node.js:** Основная серверная среда с использованием фреймворков Fastify или Express.
- **REST + WS:** Архитектура API для стандартных запросов и WebSocket для real-time взаимодействия.
- **JWT + OAuth2:** Механизмы безопасной авторизации пользователей.
- **SRS-алгоритм:** Логика интервальных повторений (Spaced Repetition System).

### База данных

Хранилища данных разных уровней.

- **PostgreSQL:** Основная реляционная база для хранения информации о пользователях и словах.
- **Redis:** Быстрое in-memory хранилище для сессий и кэша.
- **SQLite:** Локальная база данных, обеспечивающая работу приложения на устройстве в офлайн-режиме.

### Внешние сервисы

Интеграции со сторонними платформами для расширения функционала.

- **Google TTS:** Синтез речи (Text-to-Speech) для генерации правильного произношения.
- **Firebase:** Система отправки push-уведомлений.
- **Apple / Google:** Провайдеры для быстрой сторонней авторизации.
- **Stripe:** Платежный шлюз для обработки покупок (оплата тарифа Pro).

### Безопасность

Меры по защите данных и серверов.

- **HTTPS / TLS 1.3:** Надежное шифрование всего проходящего сетевого трафика.
- **Rate limiting:** Ограничение количества запросов для защиты системы от брутфорс-атак.

### Инфраструктура

Среда для развертывания и поддержания работы приложения.

- **Docker + CI/CD:** Контейнеризация сервисов и автоматизация процессов сборки/развертывания с помощью GitHub Actions.
- **AWS / Railway:** Провайдеры облачного хостинга, обеспечивающие работу и масштабирование проекта.

## Детальное описание стека

### Frontend — React Native + React (PWA)

Одна кодовая база покрывает iOS, Android и веб через React Native + React Native Web. Это дешевле и быстрее, чем делать три отдельных приложения.

**Ключевые библиотеки:**

- `Zustand` — лёгкий стейт-менеджер (проще Redux, лучше контекста)
- `React Query (TanStack)` — кэширование запросов, синхронизация в фоне, retry при потере сети
- `React Native Reanimated` — плавные 60fps анимации переворота карточек
- `expo-av` — воспроизведение аудио произношения
- `react-native-svg` — рисование иероглифов пальцем (канва для черт)
- `WatermelonDB` — офлайн-база на устройстве с синхронизацией (SQLite под капотом)

### Backend — Node.js + Fastify

Fastify в 2 раза быстрее Express при том же коде и имеет встроенную валидацию схем через JSON Schema / Zod.

**Структура API:**

```
/auth        → регистрация, вход, refresh токенов
/words       → CRUD слов, словари, тематические наборы
/sessions    → SRS-сессии, сохранение прогресса
/stats       → статистика, стрик, XP
/sync        → дельта-синхронизация с устройством
```

**SRS-алгоритм** — реализуется на сервере (алгоритм SM-2 или FSRS): хранит для каждой пары пользователь+слово интервал, ease factor и дату следующего показа. Это не бизнес-логика на клиенте, а серверная — чтобы прогресс не зависел от устройства.

### База данных

**PostgreSQL** — основное хранилище. Ключевые таблицы: `users`, `words`, `user_word_progress` (SRS-данные), `sessions`, `decks`.

**Redis** — сессии, refresh-токены, очереди уведомлений, rate-limit счётчики. Данные с TTL, не критичные для персистентности.

**SQLite на устройстве** — офлайн-режим. Пользователь занимается без интернета, при подключении — синхронизация дельты с сервером.

## Функции безопасности

**Аутентификация:**

- Короткоживущие Access JWT (15 мин) + долгоживущие Refresh токены (30 дней) в HttpOnly cookie — XSS не может украсть
- Хеширование паролей через `bcrypt` (cost factor 12) или `argon2id`
- OAuth2 через Google / Apple Sign In как альтернатива паролю

**Защита API:**

- Rate limiting через Redis: не более 5 попыток входа с одного IP за 15 минут, потом блокировка
- CORS — только разрешённые домены
- Helmet.js — автоматически выставляет правильные HTTP-заголовки безопасности
- Валидация всех входящих данных через Zod — SQL-инъекции невозможны при использовании ORM (Prisma)

**Данные:**

- Весь трафик только через TLS 1.3
- Персональные данные минимальны: email, прогресс. Никакие платёжные данные не хранятся на сервере — только через Stripe
- GDPR: кнопка «Удалить мой аккаунт» физически удаляет все данные из БД

## Вспомогательные функции для удобства

**Офлайн-режим** — WatermelonDB синхронизирует только изменения (дельта-синк). Открыл самолёт-режим → занимаешься → приземлился → всё синхронизировалось автоматически.

**Push-уведомления** — Firebase Cloud Messaging. Сервер за 30 минут до оптимального времени повторения отправляет тихий пуш. Расписание вычисляется на основе SRS и истории активности пользователя.

**Фоновая синхронизация аудио** — произношения слов скачиваются заранее (Google TTS или Forvo API) и кешируются локально. Звук играет мгновенно, без задержки сети.

**A/B-тесты** — флаги фич через LaunchDarkly или самописный механизм. Позволяет тестировать разные алгоритмы показа слов или UI без релиза.

**Аналитика** — только через самохостинг PostHog (GDPR-compliant). Отслеживаем: retention, сколько слов выучено за сессию, где пользователи уходят.

## Почему именно этот стек

| Критерий             | Выбор           | Причина                                                   |
| -------------------- | --------------- | --------------------------------------------------------- |
| Одна кодовая база    | React Native    | iOS + Android + Web из одного репо                        |
| Скорость сервера     | Fastify         | В 2× быстрее Express, JSON Schema валидация               |
| Офлайн               | WatermelonDB    | Специально спроектирована для мобильных офлайн-приложений |
| Синхронизация        | Redis + дельта  | Не гоняем весь словарь по сети каждый раз                 |
| Безопасность токенов | HttpOnly cookie | Защита от XSS-кражи токенов                               |
